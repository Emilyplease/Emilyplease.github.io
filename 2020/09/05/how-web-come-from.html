环境依赖

使用Github Pages和jekyll搭建网站；优点是免费和有很多模板可以使用；

Github Pages

Github Pages是Github提供的静态网站托管服务，可以免费托管静态网站；

jekyll

jekyll相当于是一个Ruby Gem组件, 所以安装过程需要安装Ruby;

安装过程

基于windows搭建；

安装Ruby

最麻烦的就是环境配置：

  1. 官网下载RubyInstaller， 使用默认配置项安装;官方推荐新手安装Ruby+Devkit 3.2.X (x64)
  2. ridk install , 用于安装gems；选择第三个：MSYS2 and MINGW development tool chain.
  3. 添加bin目录到环境变量中，建议重启生效；cmd执行gem install jekyll bundler

此时就可以使用jekyll自行生成blog了； 本地实时看到修改效果，需要绑定远端 bundle exec jekyll serve或者jekyll server

问题和解决

’’’ 由于我使用了模板，在build过程中遇到了很多包解析不到的问题 ‘’’

  1. 主要是参考文章安装几个包；可以自检；另外make版本不一样应该没问题；
  2. 遇到gem install一些版本依赖问题，也是Google解决；比如ffi安装错误
  3. 关于Ruby版本问题，最开始安装3.3运行失败，改为3.2就成功了，参考
  4. 解析包丢失问题，执行bundle add webrick（模块名），然后bundle install
